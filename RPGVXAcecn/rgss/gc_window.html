Window

游戏窗口的类。在内部由多重的精灵所组成。

超类

  • Object

类方法

Window.new([x, y, width, height]) (RGSS3)

创建一个窗口，并可以用参数来指定位置与大小。

方法

dispose

释放窗口。若窗口已释放则什么都不做。

disposed?

当窗口已释放则返回 true。

update

刷新光标的闪烁和暂停标记的动画。原则上每一帧都会调用一次。

move(x, y, width, height) (RGSS3)

重新设定窗口的位置与大小。

open? (RGSS3)

如果窗口被成功打开（不透明度为255），则返回 true 。

close? (RGSS3)

如果窗口被成功关闭（不透明度为0），则返回 true 。

属性

windowskin

引用一个位图 (Bitmap) 作为窗口的外观。

外观规格和 RMVX 基本相同，具体规格请参见 窗口外观的具体规格。

contents

引用作为窗口内容的位图（Bitmap ）。

cursor_rect

光标存放区域（Rect ）。

根据窗口内容，创建一个含有坐标的矩形对象。 (RGSS3)

viewport

引用与窗口关联的显示端口（Viewport ）。

active

光标闪烁的状态。true 表示闪烁。默认值为True。

visible

窗口可见的状态。true 表示可见。默认值为True。

arrows_visible (RGSS3)

光标可见的状态，true表示可见。默认值为True。

pause

暂停标记可见的状态。true 表示可见。默认值为False。

x

窗口的 X 坐标。

y

窗口的 Y 坐标。

width

窗口的宽度。

height

窗口的高度。

z

窗口的 Z 坐标。数值愈大的平面愈靠近玩家。Z 坐标相同的，最后生成的对象则会最靠近玩家。

默认值为100。 (RGSS3)

ox

窗口内容的 X 坐标。滚动窗口时修改此数值。

也可作用于光标。 (RGSS3)

oy

窗口内容的 Y 坐标。滚动窗口时修改此数值。

也可作用于光标。 (RGSS3)

padding (RGSS3)

窗口框架与内容之间的虚空区域宽度，默认值为12。 (RGSS3)

padding_bottom (RGSS3)

窗口底部的填充物。必须在设置 padding 后才可使用，因为这二者是联动的。

opacity

窗口的不透明度（0～255）。超出此范围的数值会自动修正。

back_opacity

窗口背景的不透明度（0～255）。超出此范围的数值会自动修正。默认值为192 (RGSS3)。

contents_opacity

窗口内容的不透明度（0～255）。超出此范围的数值会自动修正。默认值为255。

openness

窗口的开启程度（0～255）。超出此范围的数值会自动修正。

将此数值从 0（完全关闭）至 255（完全开启）之间变换，可以产生窗口打开和关闭的动画效果。 openness 若低于 255，则不会显示其内容。默认值 255。

tone (RGSS3)

窗口的背景颜色（Tone ）。
